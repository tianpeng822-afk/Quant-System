个人基金量化管理系统 (MyFund-Quant-System)



产品需求文档 (PRD) v3.0 (全自动化推送版)



一、 系统定位



一款面向有投资经验的个人开发者，专注“场外基金/ETF”的资产记账、多策略分析、风控预警、以及基于大模型的同类标的优选替换的全自动轻量级辅助决策系统。

原则：核心数据私有化部署，系统每日盘后自动运算，通过推送服务被动接收决策建议，无需人工主动查询。



二、 核心功能模块规划



模块 1：多账户资产记账中心 (Portfolio Tracker)



流水记账： 记录每一笔申购（含手续费）、赎回、分红（现金/红利再投），支持多账户（个人、家人）隔离与合并统计。



精准核算： 基于流水自动计算 FIFO/摊薄成本、每日绝对盈亏、整体投资组合的 XIRR（内部收益率）。



回撤追踪： 记录每只持有基金的历史最高净值，实时计算当前回撤幅度。



模块 2：行情与数据采集引擎 (Data Scraper)



净值同步： 每天 22:30 自动拉取持有基金的当日实际净值。



基准对比： 同步抓取宽基指数（如沪深300、纳指100）作为收益率基准对比参考。



全市场优选池数据： 定期（如每周）抓取全市场同类基金的近期收益、最大回撤、规模等量化指标，作为 AI 选基的数据源。



模块 3：策略与风控引擎 (Signal Engine)



个性化卖点/风控预警（硬规则）：



止盈触发： 某只基金浮盈达到预设目标（如 20%、30%），触发阶梯止盈信号。



止损/回撤触发： 当近期回撤幅度超过设定阈值（如 -15%），或连续 N 天跑输业绩基准，触发预警。



动态资产再平衡： 监控大类资产（美股、A股、固收）占比，偏离目标权重超过 5% 时给出调仓金额提示。



模块 4：DeepSeek AI 首席分析师 (AI Analyst)



弱势基金淘汰与优选推荐 (Screener)：



触发机制： 当某基金触发模块3的“止损/回撤预警”，或主动要求系统体检时触发。



数据清洗： 系统从模块2提取该基金所属大类（如：医疗主题）中，按夏普比率/超额收益排名前 20 的基金数据。



AI 决策： DeepSeek 根据这批候选基金的历史业绩稳定性、最大回撤等指标，输出 Top 3 推荐替代池，并给出明确的替换理由，供下一阶段定投参考。



板块走势诊断： 定期让 AI 总结近期涨跌幅居前板块的核心逻辑，判断是否值得加仓或定投。



模块 5：自动化定时推送 (Automated Notification)



纯单向推送机制 (Cron Job)： 每天 22:30，流水线任务跑完后，向用户发送结构化图文消息。



推送内容格式：



💰 账户概览： 今日总资产、日内盈亏、累计盈亏。



🚨 风控动作： \[提示] 基金A已达 20% 止盈线；\[警告] 基金B近期回撤达 15%。



🤖 AI 换基建议： 针对触发预警的基金B，推荐定投替代基C（附简要理由）。



三、 系统开发思路与数据流向



系统采用典型的 ETL + 策略分发 架构，所有的计算都在后台定时任务中静默完成。



核心数据流转（每日 22:30 触发）：



\[Extract] 数据抓取： 爬虫脚本运行，从第三方接口获取当天最新净值和指数行情。



\[Transform] 数据计算： 遍历本地数据库的持仓记录，用最新净值更新当前总市值、计算单只基金盈亏率和当前回撤幅度。



\[Analyze] 策略判断： 将计算结果输入规则引擎。如果指标越界，生成 Action 信号（止盈/止损/再平衡）。



\[AI Process] 大模型介入： 如果产生止损信号，触发数据采集引擎获取同类优秀基金列表，打包成 Prompt 发给 DeepSeek API，等待返回推荐结果。



\[Load \& Push] 持久化与推送： 将当天的计算快照（总资产、各基金状态）存入 SQLite 日报表，并将整理好的报告通过 Webhook 发送至手机。



四、 技术栈选型



核心语言与框架： Python 3.10+ (处理时序金融数据和调用 AI 接口的最优解)。



数据存储 (Database)： SQLite3。配合 SQLAlchemy 或 Peewee ORM 框架进行操作，单文件存储，无服务器负担，极易备份。



数据源获取 (Data Source)：



AkShare: 强大的开源金融数据接口，用于获取场外基金历史净值、同类排行等。



requests + BeautifulSoup: 备用抓取天天基金网页数据。



AI 接口引擎： DeepSeek API（调用其 deepseek-chat 模型，推理成本极低，逻辑分析能力强）。



任务调度 (Scheduler)： Linux 系统的 Crontab，或 Python 内部的 APScheduler 定时框架。



消息推送服务：



方案 A (微信)：使用 Server酱 或 PushPlus，直接将文本推送到个人微信公众号。



方案 B (钉钉/飞书)：使用自定义机器人的 Webhook，适合发送 Markdown 格式的富文本报告。



五、 实施路径规划



Phase 1: 核心资产记账与账单推送 (MVP - 基础闭环)



设计 SQLite 的账户表、资产表、流水表。



编写脚本同步净值，计算盈亏。



对接推送服务，每天晚上收到一条最基础的“今日赚了多少钱”的微信消息。



Phase 2: 规则引擎与自动化风控预警



增加计算最大回撤和跟踪止盈止损的函数。



当条件触发时，推送消息中附加风控警告。



Phase 3: 接入 DeepSeek 与自动化选基



利用 AkShare 抓取同类基金的量化指标排行。



编写 Prompt 模板，调用 DeepSeek API，解析返回的 JSON/文本结果，整合进每日推送报告中。
